O Problema

Prospecção no escuro, reativação no improviso

Cada vendedora fazia prospecção manualmente — pesquisando no Google, sem padronização, sem documentação, sem histórico. Dois vendedores podiam abordar o mesmo cliente sem saber. Reativação de inativos seguia a mesma lógica: a supervisora enviava uma planilha e torcia para não haver conflito.

"Eu quero ver nossos clientes no mapa. Ao selecionar um, quero que apareçam os possíveis leads num raio de 600m."
— Diretoria Executiva · Grupo Sales
O Desafio Técnico

Quatro regras que transformaram dados brutos em leads limpos

01
Filtro de Carteira Atual
Anti-join (NOT IN) comparando CNPJs da Receita com a base de clientes ativos. Nenhum cliente já existente aparece como lead.
02
Regra de Ouro — Higienização
Telefone repetido +3x: verifica o CNAE. Se não for contabilidade (6920-6/01), exclui. Se for, mantém — é o dono do escritório.
03
Agrupamento Matriz / Filiais
Agrupa pela raiz do CNPJ (8 dígitos). Mesma raiz + mesmo município = 1 lead. Mesma raiz + municípios diferentes = leads independentes.
04
Geolocalização via CEP
Sem lat/lon na base, converteu-se o CEP em coordenadas via BrasilAPI + Nominatim, com cache, retry exponencial e rate limiting de 1.2s.
Implementação

O algoritmo do raio circular

Com coordenadas calculadas, o filtro usa Pitágoras (a² + b² ≤ r²) para eliminar os leads que caem fora do círculo de 600m — mais eficiente que Haversine para curtas distâncias.

JavaScript — Filtro de Raio 600m
// 600m convertido para graus — pré-calculado para performance
const RAIO_QUADRADO = 0.0054 * 0.0054;

function filtrarCirculo(listaBruta, latMin, latMax, lngMin, lngMax) {
  const centroLat = (latMin + latMax) / 2;
  const centroLng = (lngMin + lngMax) / 2;

  return listaBruta.filter(empresa => {
    const dLat = empresa.latitude  - centroLat;
    const dLng = empresa.longitude - centroLng;
    // a² + b² ≤ r²  →  dentro do círculo exato
    return (dLat * dLat + dLng * dLng) <= RAIO_QUADRADO;
  });
}

A Sacada — Não Pedida, Mas Necessária

Sistema de Carteira + CRM

Ter uma lista de leads sem monitoramento é metade do problema resolvido. A diretoria não pediu — mas identifiquei que sem acompanhamento, a ferramenta seria subutilizada. Propus e co-desenvolvi o sistema de carteiras integrado.

🗂
Carteira por Vendedora
A supervisora direciona leads para cada vendedora. Elimina o problema de duas vendedoras abordando o mesmo cliente simultaneamente.
📊
Painel da Supervisora
Dashboard com KPIs de cada equipe — separado por supervisora, já que a empresa tem múltiplas equipes comerciais independentes.
⏱
Expiração Automática
Lead não convertido em 30 dias sai da carteira. A vendedora pode solicitar extensão — aprovada diretamente no painel da supervisora.
📝
Registro de Tratativas
A vendedora registra cada tentativa de contato com anotações e marcos importantes da negociação — histórico completo por lead.

Aprendizados

O que esse projeto ensinou

1
Dados públicos podem substituir APIs caras
A Receita Federal disponibiliza um dos maiores repositórios de dados empresariais do mundo — gratuitamente. A chave é a engenharia de limpeza, não o acesso.
2
Produto bom é produto que fecha o ciclo
Gerar leads sem monitoramento é desperdício. O sistema de carteiras transformou uma ferramenta de busca num sistema de gestão comercial completo.
3
Restrição orçamentária gera criatividade real
O constraint da API do Google forçou uma solução melhor — mais barata, mais controlada e com dados mais ricos do que a alternativa original.
