O Problema

Um relatório de fechamento que consumia um dia inteiro

Não existia nenhuma métrica de parametrização de custo-faturamento unificada no Grupo Sales. A única forma de obter os números de fechamento era cruzar manualmente relatórios em Excel com dados exportados do ERP — um processo que, dependendo da complexidade do mês, podia consumir um dia inteiro de trabalho de uma pessoa.

Além do custo de tempo, havia risco de erro humano em cada triangulação, ausência de comparativo histórico e total dependência de disponibilidade para que a diretoria tivesse acesso aos números.

"Precisamos de um painel único que mostre tudo — faturamento, carga, frete e remessas — com comparativo mensal e metas. Em tempo real."
— Demanda interna · Grupo Sales · Diretoria
A Trajetória

De Power BI para HTML — e por quê

Processo anterior
Relatórios manuais em Excel
Dados exportados do ERP manualmente
Triangulação manual entre planilhas
Até 1 dia para gerar um fechamento
Sem comparativo histórico
Sem metas dinâmicas
Solução atual
Dashboard HTML no intranet da empresa
Conexão direta ao banco de dados SQL
Carregamento em menos de 5 segundos
Comparativo mensal automático
Metas configuráveis via modal
Análise estratégica com alertas automáticos

A primeira versão foi construída no Power BI — mas a ferramenta apresentava limitações de integração com o intranet e dependência de licença para acesso. A decisão foi migrar para HTML puro com queries SQL diretas, hospedado dentro do próprio intranet. Isso eliminou fricções de acesso, reduziu custo e garantiu que qualquer gestor pudesse consultar os números em qualquer dispositivo da rede.

Configuração Dinâmica

Modal de metas — sem alterar código

Uma das decisões de produto mais importantes foi criar uma modal de configuração onde o gestor pode inserir as metas do período sem depender de nenhum desenvolvedor. Qualquer alteração de meta é refletida imediatamente em todos os KPIs e alertas do painel.

Aprendizados

O que esse projeto ensinou

1
A ferramenta certa depende do contexto, não do hype
Power BI é excelente — mas não era a ferramenta certa para este contexto. A migração para HTML no intranet resolveu problemas de licença, acesso e latência que o BI não conseguia endereçar.
2
Dar autonomia ao gestor é tão importante quanto o dado
A modal de metas eliminou a dependência técnica para ajustes de período. O gestor insere as metas e o painel recalcula tudo — sem ticket, sem espera.
3
Confiabilidade é o primeiro KPI de um dashboard
Antes, os números dependiam de quem fez a triangulação no dia. Agora vêm diretamente do banco — mesma fonte, sem intermediários, sem margem de erro humano.
